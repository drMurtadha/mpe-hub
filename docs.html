Hasil pembelajaran

Peserta boleh memetakan borang kepada data, membina antara muka responsif, menyediakan API Apps Script, menerbitkan GitHub Pages serta menilai privasi dan penyelenggaraan.

02

GitHub Pages menghos fail statik. Pelayar menghantar JSON kepada Apps Script. Apps Script menulis ke Google Sheets dan Google Drive sebagai pemilik deployment.

03

Takrif keperluan dahulu

Buku Log

Kehadiran, masa, aktiviti, status, lampiran.

KEW.PA-9

Pemohon, tujuan, tempat, aset, status.

MCCB

Unit, keadaan ujian, arus, trip time, pengesahan.

  1. Sahkan borang sumber dan istilah dengan pemilik proses.
  2. Tentukan medan wajib, pilihan, terbitan dan jenis data.
  3. Tentukan siapa mengisi, menyemak dan membetulkan rekod.
  4. Putuskan akses, retensi, sandaran dan polisi lampiran.

05

Sediakan Google Apps Script

  1. Buka script.google.com dan cipta projek.
  2. Salin apps-script/Code.gs.
  3. Pilih Deploy → New deployment → Web app.
  4. Tetapkan Execute as: Me dan pilih tahap akses.
  5. Benarkan Sheets dan Drive.
  6. Salin URL /exec ke config.js.
window.MPE_CONFIG = {
  GOOGLE_SCRIPT_URL: "https://script.google.com/macros/s/…/exec",
  APP_NAME: "MPE Hub"
};
FungsiPeranan
ensureSetupMencipta/membuka spreadsheet, tab dan folder.
saveRecordLock, jana ID dan tambah baris.
saveAttachmentBase64 kepada fail Drive.
getDashboardSummaryMetrik dan aktiviti terkini.
06

Google Sheets dan Drive

Spreadsheet mempunyai tab logbook, asset dan mccb. Satu rekod ialah satu baris. Senarai aset, peralatan dan bacaan disimpan sebagai JSON. Lampiran Buku Log disimpan dalam folder Drive dan URL fail ditulis ke helaian.

Sandaran

Eksport XLSX/CSV berkala, gunakan sejarah versi Drive, simpan kod Apps Script dalam repositori dan dokumentasikan pemilik sistem.

09

Keselamatan dan privasi

GitHub Pages dan kod klien boleh dilihat umum. Google Sheets dan folder Drive boleh kekal private. Endpoint “Anyone” berpotensi menerima rekod daripada pihak luar.

  • Hadkan deployment kepada domain Workspace apabila sesuai.
  • Untuk aplikasi awam, tambah autentikasi, rate limit dan CAPTCHA.
  • Hadkan MIME/saiz lampiran dan gunakan polisi retensi.
  • Gunakan akaun fungsi atau storan pasukan untuk kesinambungan.
